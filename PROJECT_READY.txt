🚀 DEPLOYMENT OPTIONS

Option 1: Production Deployment (Recommended)
--------------------------------------------
On Debian 11+ / Ubuntu 22+ server:

  sudo ./scripts/install.sh

What it does:
  ✓ Installs all dependencies (nginx, dhcp, tftp, nfs, python, node)
  ✓ Creates system user and directories
  ✓ Sets up Python virtual environment
  ✓ Installs backend dependencies
  ✓ Builds frontend
  ✓ Configures Nginx
  ✓ Creates systemd services
  ✓ Starts everything

Access: http://<server-ip>


Option 2: Development Testing (Windows/Local)
--------------------------------------------
Backend (needs Python 3.11+):
  cd backend
  python -m venv venv
  venv\Scripts\activate
  pip install -r requirements.txt
  python run.py
  → http://localhost:5000/docs

Frontend (ready now):
  cd frontend
  npm run dev
  → http://localhost:3000


Option 3: Docker-Free Cloud Deploy
----------------------------------
Any cloud VPS with Debian/Ubuntu:
  1. SSH to server
  2. Clone repo
  3. Run: sudo ./scripts/install.sh
  4. Done!
